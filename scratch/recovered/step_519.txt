6: 
7: > [!IMPORTANT]
8: > **Centralized Mode Management**: The system will transition to a strict Single-Active-Mode model. Prioritized gestures determine the active mode (`BRIGHTNESS` > `VOLUME` > `SCROLL` > `CLICK` > `CURSOR`). A mode switching cooldown of `300ms` will be enforced to prevent rapid flipping and cursor jitter.
9: 
10: > [!NOTE]
11: > **Windows Brightness Control**: We will use the `screen-brightness-control` library for direct Windows system brightness integration. If it fails to load or run (e.g. in VM/headless environments), it will fall back to simulated brightness adjustments gracefully.
